# Copy Fail攻撃試してみた

非常に簡単に再現できてしまったので載せます。ＧＷ中に最悪ですね・・・

## 参考サイト

[Copy Fail](https://copy.fail/)

## 検証

今回はAzureの仮想マシンにUbuntu 24.04 LTSを入れて起動しています。
SSHで接続後に一般ユーザーでエクスプロイト用コードを実行してみます。

エクスプロイト用コードはこんな感じ

実行してみる
実行後に `id` コマンド打鍵したらrootに昇格していました。ひえぇ。

```bash
ubuntu@ubuntu:~$ python3 exp.py
# id
uid=0(root) gid=1000(ubuntu) groups=1000(ubuntu),4(adm),24(cdrom),27(sudo),30(dip),105(lxd)
```

これは `/usr/bin/su` を書き換えるらしいので、エクスプロイト用コードの差分を確認してみると、確かにバイナリに差分がありました。

```bash
ubuntu@ubuntu:~$ cmp su_before su_after
su_before su_after differ: byte 17, line 1
```

影響ある場合はパッチ当てるなりなんなりしないとまずいですね。
