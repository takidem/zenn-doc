## モチベーション

最近Pulumiを触るようになったので、AWS環境でPulumiを安全に実行できるようにする開発環境を整えたいというモチベ

## 注意

- WSL2環境(Ubuntu-24.04)を使ってます
- 筆者はPulumiビギナーです
- 本記事はPulumiですが、応用すればTerraformでもできます

## 利用するもの

- awscli
- aws-vault
- pass
- pulumi

1. 生成したGPGキーでpassストアを作成する

```bash
$ GPG_PUB_KEY=73F9858F13E6D7A5BA14CD64D728CA13483C82EC

$ pass init $GPG_PUB_KEY
mkdir: created directory '/home/takidem/.password-store/'
Password store initialized for 73F9858F13E6D7A5BA14CD64D728CA13483C82EC

# この時点ではパスワードストアができただけ
$ pass
Password Store

# bashrcにaws-vaultの環境変数を追加する
$ vim ~/.bashrc
export AWS_VAULT_BACKEND=pass
export AWS_VAULT_PASS_PREFIX=aws-vault
export AWS_SESSION_TOKEN_TTL=3h
export GPG_TTY=$(tty)

# 適用させる
source ~/.bashrc
```

## aws-vaultのセットアップ

以下のコマンド打鍵

```bash
# この時点では ~/.aws/configはカラ
$ cat ~/.aws/config

# MFAは人権なのでMFAしっかりと有効化してアクセスキーを発行しておく
# IAMユーザーのソースプロファイルを保存する
$ aws-vault add default
Enter Access Key ID: AKIA3UL23SQIM23P37F3
Enter Secret Access Key: ****************************************
Added credentials to profile "default" in vault

# このタイミングでpassストアを確認するとプロファイル名で認証情報ができていることがわかる
$ pass
Password Store
└── aws-vault
    └── default

# configを編集してregion, output, MFAデバイスのARNを指定する
# MFAデバイスのARNはマネコンで確認可能
$ vim ~/.aws/config
[default]
region = ap-northeast-1
output = json
mfa_serial = arn:aws:iam::XXXXXXXXXXXX:mfa/takidem

# aws-vaultの一覧を確認する
$ aws-vault ls
Profile                  Credentials              Sessions
=======                  ===========              ========
default                  default                  -

# aws-vaultのinvoke用コマンドでaws cliを呼び出す
$ aws-vault exec default -- aws sts get-caller-identity

    "UserId": "AIDA3UL23SQIIC5GUDRVA",
    "Account": "XXXXXXXXXXXX",
    "Arn": "arn:aws:iam::XXXXXXXXXXXX:user/takidem"
```
