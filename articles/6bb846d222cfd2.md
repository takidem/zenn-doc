# Azure app service domains で取得済のドメインを用いて acme 自動更新対応の証明書を整備する

## モチベーション

自宅サーバの環境で長らく使っていた Duck DNS で管理しているドメインの証明書をそろそろ変えたいなという動機です。
証明書を使用するのは、 Proxmox 上の LXC で稼働している Nginx Proxy Manager(以下NPM) です。
今回は NPM 使えば大したことしていないので、こんなことしたなぁという記録用です。

## 自宅サーバの NW 構成

こんな感じの構成を取っています。
本当は Tailscale には不満があるので使いたくないのですが、リモートで SSH したいとかがあり今後も使っていくことになると思います。
Kubernetes Cluster は今回登場しません。面白いことできたら記事書こうと思います。

![](/images/6bb846d222cfd2/001.png)

Tailscale は、 `resolve.conf`を勝手に書き換えて、強制的に `100.100.100.100`にします。
名前解決がうまくいかなかったり、タイムアウトしたり結構ストレスが溜まります。
後は iptables 周りもひどく、 Tailscale は `100.64.0.0/10`を予約しているのですが、この範囲への通信を強制的に `DROP`します。
Tailscale は非常に導入が簡単ですが、いろいろ気になる部分も多く、早く脱却したいと思う今日この頃です。

## 構築

ACME については[こちら](https://jprs.jp/pubcert/about/ACME/)を確認ください。

NPM の ACME クライアントが自動的に Azure DNS に検証用レコードを追加できるようにサービスプリンシパルを発行する必要があります。
また、今回は `takidemdev.net`をそのまま使うのではなく、サブドメインのワイルドカードで証明書を取って行きます。
したがって、以下のような流れで構築作業をしていきます。

1. 子ゾーン作成/DNS 委任設定
1. `DNS Zone Contributor`ロールを付与したサービスプリンシパルの作成
1. NPM 上で証明書の発行

### 子ゾーン作成

azcliを使用します。

```bash
# 変数定義
RG="<resourceGroupName>"
PARENT_ZONE="takidemdev.net"
## 今回はhomeという子ゾーンを作ります
CHILD_ZONE="home.takidemdev.net"

# az login済みです

# 子ゾーン作成
az network dns zone create \
  --resource-group $RG \
  --name $CHILD_ZONE

# 親ゾーンに委任用レコードを作成
az network dns record-set ns create \
  --resource-group $RG \
  --zone-name $PARENT_ZONE \
  --name home

# レコード追加(４つあるのでそれぞれ追加します)
az network dns record-set ns add-record \
  --resource-group $RG \
  --zone-name $PARENT_ZONE \
  --record-set-name home \
  --nsdname <NS record>
```

### `DNS Zone Contributor`ロールを付与したサービスプリンシパルの作成

子ゾーンに対して、DNS 検証用の TXT レコードを API 経由で追加するので、子ゾーンに対する `DNS Zone Contributor`権限が必要です。

```bash
az ad sp create-for-rbac \
  --name "certbot-dns-azure" \
  --role "DNS Zone Contributor" \
  --scopes /subscriptions/<subscriptionId>/resourceGroups/<resourceGroupName>/providers/Microsoft.Network/dnszones/home.takidemdev.net
```

正常に作成できると `Application Id`, `client secret` が発行されるので、控えておきましょう、

### NPM 上で証明書の発行

NPM の WebUI 上で発行します。
ホーム画面より、[Certificate]>[Add Certificate]を押下します。

![](/images/6bb846d222cfd2/002.png)

[Let's Encrypt via DNS]を選択します。
DNS 検証を通して、 Let's Encrypt で証明書を発行する方法です。

![](/images/6bb846d222cfd2/003.png)

基本設定は以下のようにします。
Domain Name については、今回ワイルドカードで取得するので、このようにしています。

| 設定項目 | 設定値 |
| -------------- | --------------- |
| Domain name | home.takidemdev.net, *.home.takidemdev.net |
| Key Type | ECDSA 256 |
| DNS Provider | Azure |

最後に Credentials File Content を指定する必要がありますが、今回は Azure を使用しているので、`Using Service Principal`及び`Zones`を記入する必要があります。
zone1,2には `FQDN:<親・子ゾーンのリソースID>`を指定します。

| 設定項目   | 設定値    |
|--------------- | --------------- |
| dns_azure_sp_client_id   | <Application Id> |
| dns_azure_sp_client_secret   | <client secret>  |
| dns_azure_tenant_id   | <Azure Tenant Id>   |
| zone1   | takidemdev.net:/subscriptions/<subscriptionId>/resourceGroups/<resourceGroupName>/providers/Microsoft.Network/dnszones/takidemdev.net   |
| zone2   | home.takidemdev.net:/subscriptions/<subscriptionId>/resourceGroups/<resourceGroupName>/providers/Microsoft.Network/dnszones/home.takidemdev.net   |

![](/images/6bb846d222cfd2/004.png)

入力が完了したら右下の Save を押下します。
正常に証明書を発行できると一覧に証明書が表示されます。

![](/images/6bb846d222cfd2/005.png)

## 動作確認

Proxmox 上でホストし、 NPM で公開しているサービス(NPM 自身)の一つに対して今回新しく発行した証明書を適用してみます。
現在は別の証明書が割り当たっています。

![](/images/6bb846d222cfd2/006.png)

こちらを新しい証明書に入れ替えて、サブドメインを割り当てます。

![](/images/6bb846d222cfd2/007.png)

実際にアクセスするための DNS の設定をします。
Tailscale 経由で NPM にアクセスをしているので、 Azure DNS の子ゾーンには Tailscale で NPM に割り当たっている IP アドレスを A レコードで設定する必要があります。
では、 Tailscale の管理画面を見てみます。

![](/images/6bb846d222cfd2/010.png)

Tailscale 上で `100.89.203.13`という IP アドレスが割り当たっているので、Azure DNS にこのレコードを IP アドレスを引けるように設定をします。

![](/images/6bb846d222cfd2/011.png)

では、実際にアクセスをしてみます。

![](/images/6bb846d222cfd2/012.png)

はい、新しい証明書で SSL 終端できました。
後は煮るなり焼くなり好きにできますね。
