## モチベーション

実機にminikubeなんて入れたくないですよね？（決めつけ）
Kubernetesをローカルで検証したい時に便利なようにKubernetesの仮想環境を用意するための資材が欲しかった。
同じように手元で簡単にKubernetes環境を作って検証して、消して、また作って・・・みたいなことをお手軽にやりたい方はご活用ください。
何か間違っているところあれば、PR出してください。

## 注意

- 以下インストール済みであることが必要
    - Vagrant
    - Oracle Virtual Box

## 構築方法

資材は私のGithubからクローンしてください。

```powershell
PS> git clone https://github.com/takidem/prac-minikube.git
Cloning into 'prac-minikube'...
remote: Enumerating objects: 42, done.
remote: Counting objects: 100% (42/42), done.
remote: Compressing objects: 100% (31/31), done.
remote: Total 42 (delta 10), reused 16 (delta 3), pack-reused 0 (from 0)
Receiving objects: 100% (42/42), 14.08 KiB | 4.69 MiB/s, done.
Resolving deltas: 100% (10/10), done.

PS> cd prac-minikube

PS> vagrant up

# しばらく待って、以下のコマンドでrunning状態だったらOK
PS> vagrant status

# 起動した仮想マシンにSSHで入りたい時は以下のコマンド
PS> vagrant ssh
Bringing machine 'minikube' up with 'virtualbox' provider...
==> minikube: Box 'bento/ubuntu-24.04' could not be found. Attempting to find and install...

```

上記のようにVagrantファイルとminikube.yamlに組み込まれているAnsibleプレイブックが正常に実行完了すればOKです。
起動するとこんな感じでVirtual Boxに仮想マシンが立ち上がります。
![](https://storage.googleapis.com/zenn-user-upload/ff3e196f6916-20260125.png)

では、次にminikubeを実行している仮想マシンにログインしてみます。
```bash
$ vagrant ssh

# kubectlのバージョンを見てみます
```

```PowerShell
PS> vagrant halt
```
